# OG Image Pipeline — peterbenoit.com

OG images are generated by the sibling repo `../OGImageGenerator` (must exist at that path).

## How It Works

- `prebuild` automatically runs `og:sync` before every `npm run build` if the generator is present
- `og:sync` reads page metadata, generates `*.png` files, copies them to `public/og/`
- `og:force` does the same but regenerates all images unconditionally
- AI generation is optional and manual-only; default behavior is deterministic template generation
- **When adding a new page:** image auto-generates on next `og:sync` or `build` — no manual work
- The `og:image` meta tag must reference: `https://peterbenoit.com/og/og-{slug}.png`
- If OGImageGenerator is absent (e.g. CI), `prebuild` prints a warning and continues — do not hard-fail

## Operator Workflow

Run from `../OGImageGenerator`:

```sh
# Dry-run (no file writes, no provider calls)
node generate-all.js --dry-run

# Dry-run with AI mode preview (prompts/seeds/cache outcomes only)
OG_AI_ENABLED=1 OG_DRY_RUN=1 node generate-all.js --dry-run

# Incremental generation
node generate-all.js

# Force regeneration
node generate-all.js --force
```

## AI Mode Env Vars (optional)

- `OG_AI_ENABLED`, `OG_AI_PROVIDER`, `OG_AI_MODEL` / `OG_OPENAI_MODEL`
- `OG_AI_RETRIES`, `OG_AI_TIMEOUT_MS`, `OG_AI_SEED_STRATEGY`
- `OG_OPENAI_API_KEY` or `OPENAI_API_KEY`

## Review Checklist Before Publish

1. Run dry-run and inspect planned outcomes
2. Confirm prompts/seeds look deterministic and sane for changed pages
3. Generate images and verify cache behavior in logs (`cache-hit` vs regenerate)
4. Spot-check readability and accent consistency on sampled outputs
5. Confirm files copied into `public/og/` and referenced by `og:image` tags
