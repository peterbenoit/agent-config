## Glossary

Terms that have a specific meaning in this library. Skip general programming terms — only the
ones that would confuse or mislead an outside reader belong here.

**{Term}** — {Definition. Include the relevant file or export if helpful.}

<!-- Example entries — replace with your own: -->

**Channel** — A named event scope. Subscribers and publishers interact only within the same
channel. Channels are created lazily on first use. Not the same as a topic (which is the
event name within a channel).

**Topic** — The event name within a channel. Wildcards are supported: `user.*` matches
`user.created` and `user.deleted`. Defined in `src/matching.ts`.

**Dead letter** — A published event for which no subscriber exists at the time of publication.
By default, dead letters are silently dropped. `{ deadLetter: true }` in channel options
queues them for late subscribers. This behavior is intentional and tested — do not remove it.

**Subscriber token** — The opaque object returned by `.subscribe()`. Required to unsubscribe.
Tokens are not serializable and must not be stored in state that persists across page reloads.

## Key Relationships

Non-obvious connections between parts of the library that affect how changes ripple through.

<!-- Example entries — replace with your own: -->

- `src/index.ts` is the public entry point. Only symbols explicitly exported here are part of
  the public API. Internal modules should not be imported directly by consumers. Any breaking
  change to `src/index.ts` exports requires a major version bump.

- `src/matching.ts` (wildcard topic matching) is imported by both `src/channel.ts` and
  `src/bus.ts`. Changes to matching behavior affect both publish and subscribe paths. Update
  tests in `test/matching.test.ts` before touching this file.

- The `dist/` directory is generated by `npm run build` and committed to git. This is
  intentional — the package is published from `dist/`, not from `src/`. Do not add `dist/`
  to `.gitignore`.

- `test/` uses Vitest with a browser environment (`@vitest/browser`). Tests that depend on
  `window` or `document` will not pass in the default Node environment. Check `vitest.config.ts`
  before adding environment-sensitive tests.

## Decisions

Choices that were made deliberately and define the library's contract.

<!-- Example entries — replace with your own: -->

**Synchronous delivery.** Event delivery is synchronous by default — subscribers are called in
the same tick as `publish()`. This is intentional and tested behavior. Do not make delivery
async to "simplify" something — it changes observable behavior for existing consumers.

**No runtime dependencies.** This library has zero runtime dependencies. Do not add a dependency
to solve something the standard library handles. If a dependency is genuinely necessary, it
requires an explicit decision and a major version bump.

**Semver is strict.** Adding a required parameter, removing an export, or changing a default
value is always a major version bump. Changing an option that defaults to the same behavior
is a patch. When in doubt: major.

**No global singleton.** There is no default export bus instance. Consumers create their own
`Bus` instances. This was a deliberate reversal from v1. Do not re-introduce a singleton.
